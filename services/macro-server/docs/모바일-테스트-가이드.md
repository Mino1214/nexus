# 📱 모바일 최적화 테스트 가이드

## ✅ 적용 완료!

홈페이지가 모바일에 최적화되었습니다!

---

## 🧪 테스트 방법

### 1️⃣ Chrome 개발자 도구로 테스트

```bash
1. Chrome에서 http://localhost:3000 접속
2. F12 누르기 (개발자 도구 열기)
3. Ctrl+Shift+M 누르기 (모바일 모드 토글)
4. 상단에서 디바이스 선택:
   - iPhone SE (작은 화면)
   - iPhone 12 Pro (중간 화면)
   - iPad (태블릿)
```

#### 확인할 것들:
- [ ] 텍스트가 잘 읽히는가?
- [ ] 버튼이 터치하기 쉬운가?
- [ ] 레이아웃이 깨지지 않는가?
- [ ] 스크롤이 부드러운가?

---

### 2️⃣ 실제 모바일 기기로 테스트

#### Step 1: IP 주소 확인
```bash
# 터미널에서
ip addr show | grep "inet " | grep -v 127.0.0.1

# 또는
hostname -I
```

#### Step 2: 모바일에서 접속
```
http://YOUR_IP:3000

예시:
http://192.168.0.10:3000
http://192.168.1.105:3000
```

#### 확인할 것들:
- [ ] 페이지가 빠르게 로드되는가?
- [ ] 버튼 터치가 정확한가?
- [ ] 폰트 크기가 적절한가?
- [ ] 화면 회전 시 정상 작동하는가?

---

### 3️⃣ 다크모드 테스트

#### iPhone
```
설정 → 화면 및 밝기 → 다크 모드
```

#### Android
```
설정 → 디스플레이 → 다크 테마
```

#### Chrome 개발자 도구
```
F12 → More tools → Rendering
→ Emulate CSS media feature prefers-color-scheme: dark
```

#### 확인할 것들:
- [ ] 다크모드에서 텍스트가 잘 보이는가?
- [ ] 배경색이 자연스러운가?
- [ ] 대비가 적절한가?

## 🐛 문제 해결

### 페이지가 안 열려요
```bash
# 서버 실행 확인
ss -tlnp | grep :3000

# 방화벽 확인
sudo ufw status

# 네트워크 확인
ping YOUR_IP
```

### 모바일에서 접속이 안 돼요
```bash
# 같은 Wi-Fi 네트워크인지 확인
# 방화벽에서 3000 포트 허용
sudo ufw allow 3000

# 서버가 0.0.0.0에서 리슨하는지 확인
ss -tlnp | grep :3000
```

### 레이아웃이 깨져요
```bash
# 캐시 삭제
- Chrome: Ctrl+Shift+R
- 모바일: 설정 → 캐시 삭제

# 또는 시크릿 모드로 테스트
```
