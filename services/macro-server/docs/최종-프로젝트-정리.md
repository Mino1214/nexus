## 🚀 핵심 기능

### 1️⃣ 회원가입/승인 시스템
```
사용자 가입 (추천인 코드)
   ↓
관리자 페이지에 "승인 대기" 표시
   ↓
매니저가 "승인" 버튼 클릭
   ↓
"사용기간 설정" (30/90/180/365일)
   ↓
로그인 가능!
```

### 2️⃣ 사용자 상태 관리
- **⏳ pending**: 승인 대기 중 (로그인 불가)
- **✅ approved**: 승인됨 (로그인 가능)
- **🚫 suspended**: 정지됨 (로그인 불가)

### 3️⃣ 세션 관리
- **24시간 슬라이딩 세션**: 활동 시 자동 연장
- **서버 재시작 시에도 유지**: MariaDB에 저장
- **자동 만료**: 24시간 동안 활동 없으면 로그아웃

### 4️⃣ 권한 시스템
- **마스터**: 모든 기능 + 매니저 관리 + 시드 조회
- **매니저**: 내 사용자만 관리 + 승인/정지/기간설정

## 📊 데이터베이스 구조

### `users` - 사용자
| 컬럼              | 타입          | 설명                          |
|-------------------|---------------|-------------------------------|
| id                | VARCHAR(50)   | 아이디 (PK)                   |
| pw                | VARCHAR(100)  | 비밀번호                      |
| manager_id        | VARCHAR(50)   | 소속 매니저 (추천인)          |
| telegram          | VARCHAR(100)  | 텔레그램 ID                   |
| status            | ENUM          | pending/approved/suspended    |
| expire_date       | DATETIME      | 만료일                        |
| subscription_days | INT           | 구독 일수                     |
| created_at        | TIMESTAMP     | 생성일                        |

### `managers` - 관리자
| 컬럼       | 타입        | 설명               |
|------------|-------------|--------------------|
| id         | VARCHAR(50) | 아이디 (PK)        |
| pw         | VARCHAR(100)| 비밀번호           |
| role       | ENUM        | master/manager     |
| created_at | TIMESTAMP   | 생성일             |

### `sessions` - 세션
| 컬럼          | 타입         | 설명                    |
|---------------|--------------|-------------------------|
| token         | VARCHAR(255) | 세션 토큰 (PK)          |
| user_id       | VARCHAR(50)  | 사용자 ID               |
| last_activity | TIMESTAMP    | 마지막 활동 시각 (갱신) |
| created_at    | TIMESTAMP    | 생성일                  |

### `seeds` - 시드 문구
| 컬럼       | 타입        | 설명           |
|------------|-------------|----------------|
| id         | INT (AI)    | ID (PK)        |
| user_id    | VARCHAR(50) | 사용자 ID      |
| seed       | TEXT        | 시드 문구      |
| created_at | TIMESTAMP   | 수신 시각      |

### `settings` - 설정
| 컬럼          | 타입         | 설명       |
|---------------|--------------|------------|
| setting_key   | VARCHAR(100) | 설정 키    |
| setting_value | TEXT         | 설정 값    |
| updated_at    | TIMESTAMP    | 수정 시각  |

## 🔌 API 엔드포인트

### 📝 사용자 API (인증 불필요)
```http
POST /api/register
  body: { id, password, referralCode, telegram }
  response: { success, message, managerId }

POST /api/login
  body: { id, password }
  response: { token, role, expireDate, remainingDays }

POST /api/validate
  body: { token }
  response: { valid, id }

POST /api/logout
  body: { token }

POST /api/telegram
  response: { nickname }
```

### 🔐 관리자 API (인증 필요)
```http
POST /api/admin/login
  body: { id, pw }
  response: { success, role }

GET /api/admin/pending-users
  response: [{ id, managerId, telegram, createdAt, status }]

POST /api/admin/approve-user
  body: { userId }

POST /api/admin/reject-user
  body: { userId }

POST /api/admin/set-subscription
  body: { userId, days }  # 30/90/180/365

POST /api/admin/suspend-user
  body: { userId, suspend }  # true/false

GET /api/admin/users
  response: [{ id, managerId, telegram, status, expireDate, remainingDays }]

POST /api/admin/users
  body: { id, pw, managerId, telegram }  # 마스터 전용

GET /api/admin/sessions
  response: [{ token, userId }]

DELETE /api/admin/sessions/:userId

POST /api/admin/users/:id
  # 탈퇴 (DELETE)

GET /api/admin/managers
  response: [{ id, role }]

POST /api/admin/managers
  body: { id, pw }  # 마스터 전용

DELETE /api/admin/managers/:id
  # 매니저 삭제 (마스터 전용)

GET /api/admin/seeds
  response: [{ id, userId, seed, createdAt }]  # 마스터 전용

GET /api/admin/telegram
  response: { nickname }

POST /api/admin/telegram
  body: { nickname }
```

## 🔧 주요 스크립트

### MariaDB 관련
```bash
# 초기 설정 (데이터베이스, 테이블, 사용자 생성)
sudo ./scripts/setup-mariadb.sh

# 테이블 재생성 (스키마 변경 시)
sudo ./scripts/recreate-tables.sh

# 기존 파일 데이터 마이그레이션
sudo ./scripts/migrate-to-db.sh

# 원격 접속 설정
sudo ./scripts/setup-remote-access.sh

# 마스터 계정 생성
sudo ./scripts/create-master-account.sh
```

### Nginx 관련
```bash
# nexus001.vip 도메인 설정
sudo ./scripts/setup-nginx-nexus001.sh

# Nginx 설정 테스트
sudo nginx -t

# Nginx 재시작
sudo systemctl reload nginx
```

### 서버 관리
```bash
# PM2로 서버 시작
pm2 start server.js --name mynolab-server

# 서버 재시작
pm2 restart mynolab-server

# 로그 확인
pm2 logs mynolab-server

# 서버 상태 확인
pm2 status

# 부팅 시 자동 시작 설정
pm2 startup
pm2 save
```

## 🔍 문제 해결

### 서버가 시작되지 않을 때
```bash
# 로그 확인
pm2 logs mynolab-server --lines 50

# 포트 충돌 확인
sudo lsof -i :3000

# 프로세스 삭제 후 재시작
pm2 delete mynolab-server
pm2 start server.js --name mynolab-server
```

### DB 연결 오류
```bash
# MariaDB 상태 확인
sudo systemctl status mariadb

# root로 접속
sudo mysql -u root

# mynolab_user로 접속
mysql -u mynolab_user -p'mynolab2026' mynolab

# 사용자 권한 확인
SHOW GRANTS FOR 'mynolab_user'@'localhost';
```

### Nginx 오류
```bash
# 설정 테스트
sudo nginx -t

# 에러 로그 확인
sudo tail -f /var/log/nginx/error.log

# 접근 로그 확인
sudo tail -f /var/log/nginx/nexus001.vip-access.log

# Nginx 재시작
sudo systemctl reload nginx
```

### 403 Forbidden 오류
```bash
# public 폴더 권한 확인
ls -la public/

# 실행 권한 부여
chmod o+x /home/myno
chmod o+x /home/myno/바탕화면
chmod o+x /home/myno/바탕화면/myno
chmod o+x /home/myno/바탕화면/myno/macroServer
chmod o+x /home/myno/바탕화면/myno/macroServer/public
```
