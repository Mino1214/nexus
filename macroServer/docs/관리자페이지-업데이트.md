# 🎨 관리자 페이지 (admin.html) 업데이트

## ✅ 추가된 기능

### 1️⃣ 승인 대기 목록 섹션
- **위치**: 사용자 관리 위에 새로 추가
- **기능**:
  - 승인 대기 중인 사용자 목록 표시
  - 신청일 표시
  - 승인/거부 버튼

### 2️⃣ 사용자 목록에 추가된 컬럼
- **상태**: ✅ 승인, 🚫 정지, ⏳ 대기
- **사용기간**: X일 남음 / 만료됨

### 3️⃣ 새로운 액션 버튼
- **⏱️ 기간설정**: 30/90/180/365일 설정
- **🚫 정지 / ✅ 활성화**: 사용자 정지/활성화 토글
- **세션끊기**: 기존 유지
- **탈퇴**: 기존 유지

### 4️⃣ 매니저 권한 변경
- **직접 생성 불가**: 매니저는 "직접 생성" 버튼이 숨김 처리됨
- **승인만 가능**: 승인 대기 목록에서 승인/거부만 가능
- **마스터 전용**: 직접 생성은 마스터만 가능 (자동 승인됨)

## 🎯 사용 시나리오

### 1. 새 사용자 가입 후 승인
```
1. 사용자가 클라이언트에서 회원가입
   ↓
2. 관리자 페이지 "승인 대기 목록"에 표시됨
   ↓
3. 매니저가 "✅ 승인" 버튼 클릭
   ↓
4. "사용자 관리"로 이동됨
   ↓
5. "⏱️ 기간설정" 버튼으로 30/90/180/365일 설정
   ↓
6. 사용자 로그인 가능!
```

### 2. 사용자 정지
```
1. "사용자 관리"에서 해당 사용자 찾기
   ↓
2. "🚫 정지" 버튼 클릭
   ↓
3. 상태가 "🚫 정지"로 변경
   ↓
4. 해당 사용자 로그인 불가
```

### 3. 사용기간 연장
```
1. "사용자 관리"에서 "X일 남음" 확인
   ↓
2. "⏱️ 기간설정" 버튼 클릭
   ↓
3. 30/90/180/365일 중 선택
   ↓
4. 현재 시간부터 새로 계산됨
```

## 🔧 새로운 JavaScript 함수

### `loadPendingUsers()`
- 승인 대기 목록 로드
- 마스터: 전체 / 매니저: 자기 소속만

### `approveUser(userId)`
- 사용자 승인
- API: `POST /api/admin/approve-user`

### `rejectUser(userId)`
- 사용자 거부 (삭제)
- API: `POST /api/admin/reject-user`

### `setSubscription(userId)`
- 사용기간 설정
- 30/90/180/365일 중 선택
- API: `POST /api/admin/set-subscription`

### `toggleSuspend(userId, currentStatus)`
- 사용자 정지/활성화 토글
- API: `POST /api/admin/suspend-user`

### `loadUsers()` (수정됨)
- 상태 및 사용기간 표시 추가
- 새로운 액션 버튼 추가

## 🧪 테스트 방법

### 1. 마스터로 로그인
```
URL: https://nexus001.vip/admin.html
아이디: tlarbwjd
비밀번호: tlarbwjd
```

### 2. 테스트 시나리오

#### A. 승인 테스트
1. 클라이언트에서 새 사용자 가입 (추천인: qazwsx)
2. 관리자 페이지 새로고침
3. "승인 대기 목록"에서 확인
4. "✅ 승인" 클릭
5. "사용자 관리"에서 확인

#### B. 사용기간 설정
1. "⏱️ 기간설정" 클릭
2. "30" 입력
3. "X일 남음" 표시 확인

#### C. 정지 테스트
1. "🚫 정지" 클릭
2. 상태가 "🚫 정지"로 변경 확인
3. 해당 사용자로 로그인 시도 → "정지되었습니다" 에러 확인

#### D. 활성화
1. "✅ 활성화" 클릭
2. 상태가 "✅ 승인"으로 변경 확인
3. 로그인 가능 확인

## 📝 주의사항

### 사용기간 설정 시
- 현재 시간부터 새로 계산됩니다
- 기존 만료일은 무시되고 새로 설정됩니다
- 예: 2월 18일에 30일 설정 → 3월 20일까지

### 승인 시
- 승인 즉시 `status = 'approved'`로 변경
- 하지만 사용기간이 없으면 로그인 가능
- 사용기간 설정 권장!

### 정지 vs 거부
- **정지**: 계정 유지, 로그인 불가
- **거부**: 계정 삭제 (되돌릴 수 없음)
