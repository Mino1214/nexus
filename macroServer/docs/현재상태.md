## 🚀 Nginx 설정 업데이트 방법

기존 사이트를 macroServer로 교체하려면:

### 방법 1: 자동 업데이트 스크립트 (권장) ⭐

```bash
cd /home/myno/바탕화면/myno/macroServer
./update-nginx.sh
```

이 스크립트는 자동으로:
- ✅ 기존 설정을 백업합니다
- ✅ 새 설정을 적용합니다
- ✅ Nginx를 재시작합니다
- ✅ 문제 발생 시 롤백 안내를 제공합니다

### 방법 2: 수동 업데이트

```bash
# 1. 기존 설정 백업
sudo cp /etc/nginx/sites-available/mynolab.kr /home/myno/backup_mynolab.kr.$(date +%Y%m%d)

# 2. 새 설정 적용
sudo cp /home/myno/바탕화면/myno/macroServer/mynolab.kr.conf /etc/nginx/sites-available/mynolab.kr

# 3. Nginx 설정 테스트
sudo nginx -t

# 4. Nginx 재시작
sudo systemctl reload nginx
```

## 🔍 현재 서버 확인

### Express 서버 상태
```bash
# 프로세스 확인
ps aux | grep "node server.js" | grep macroServer

# 포트 확인
ss -tlnp | grep :3000

# 로그 확인
tail -f /home/myno/바탕화면/myno/macroServer/server.log
```

### Nginx 상태
```bash
# Nginx 상태
sudo systemctl status nginx

# 현재 설정 확인
cat /etc/nginx/sites-available/mynolab.kr | grep "root"

# 액세스 로그
sudo tail -f /var/log/nginx/mynolab.kr.access.log
```
