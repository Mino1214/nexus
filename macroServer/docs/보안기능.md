# 🔒 관리자 페이지 보안 기능

## 적용된 보안 기능

### 1. 콘솔 보호 ✅
- **모든 console 함수 비활성화**
  - `console.log`, `console.error`, `console.warn` 등 모두 차단
  - 개발자 콘솔에서 디버깅 불가능

### 2. API 경로 난독화 ✅
- **Base64 인코딩으로 API 경로 숨김**
  ```javascript
  // 예: '/api/admin/login' → 'L2FwaS9hZG1pbi9sb2dpbg=='
  const _path = atob('L2FwaS9hZG1pbi9sb2dpbg==');
  ```
- **네트워크 탭에서도 원본 경로만 보임** (하지만 코드에서는 숨겨짐)

### 3. Fetch 오버라이드 ✅
- **원본 fetch를 래핑하여 추적 어렵게 만듦**
- **캐시 비활성화**로 네트워크 탭 추적 방해

### 4. 개발자 도구 감지 ✅
- **화면 크기 변화 감지**
  - 개발자 도구가 열리면 감지 (경고는 주석 처리됨)
  - 필요시 주석 해제하면 경고창 표시 가능

### 5. 우클릭 방지 ✅
- **컨텍스트 메뉴(우클릭) 완전 차단**

### 6. 키보드 단축키 차단 ✅
차단되는 단축키:
- **F12** - 개발자 도구 열기
- **Ctrl+Shift+I** - 검사 도구
- **Ctrl+Shift+J** - 콘솔 열기
- **Ctrl+U** - 소스 보기
- **Ctrl+S** - 페이지 저장

## 보안 강도

### 🟢 일반 사용자
- **접근 거의 불가능**
- 우클릭 차단, 단축키 차단으로 개발자 도구 열기 어려움

### 🟡 중급 개발자
- **접근 어려움**
- 주소창에서 `javascript:` 실행 등의 우회 방법 필요
- API 경로를 찾기 어려움 (난독화됨)

### 🟠 고급 개발자
- **접근 가능하지만 시간 소요**
- 브라우저 설정에서 직접 개발자 도구 활성화 필요
- 네트워크 탭에서 여전히 요청 확인 가능
- 코드 난독화로 분석 시간 증가

## 추가 보안 옵션 (선택사항)

### 더 강력한 보호가 필요하다면:

#### 1. 전체 코드 난독화
JavaScript 난독화 도구 사용:
```bash
npm install -g javascript-obfuscator
javascript-obfuscator admin.html --output admin.min.html
```

#### 2. 서버 사이드 렌더링
- PHP나 Node.js로 HTML을 서버에서 렌더링
- 클라이언트에서 소스 코드 보기 어려움

#### 3. 타임스탬프 기반 토큰
- 매 요청마다 시간 기반 토큰 생성
- 네트워크 재생 공격 방지

#### 4. IP 화이트리스트
- 특정 IP에서만 관리자 페이지 접근 허용
- Nginx에서 설정 가능

#### 5. 2단계 인증 (2FA)
- Google Authenticator 등과 연동
- 로그인 시 추가 인증 필요

## 현재 적용 상태

### ✅ 활성화됨
- 콘솔 차단
- 우클릭 방지
- 단축키 차단
- API 경로 난독화
- 개발자 도구 감지

### 💡 선택적 활성화
개발자 도구 감지 시 경고창을 표시하려면:

`admin.html` 파일에서 다음 줄의 주석을 해제하세요:
```javascript
// alert('개발자 도구가 감지되었습니다.');
↓
alert('개발자 도구가 감지되었습니다.');
```

## 테스트 방법

### 1. 우클릭 테스트
- 관리자 페이지에서 우클릭 → 차단되어야 함

### 2. 단축키 테스트
- **F12** 눌러보기 → 아무 반응 없어야 함
- **Ctrl+Shift+I** → 아무 반응 없어야 함

### 3. 콘솔 테스트
- 브라우저 다른 방법으로 개발자 도구 열기
- `console.log('test')` 입력 → 아무 출력 없어야 함

### 4. 네트워크 테스트
- 개발자 도구 > Network 탭
- 로그인 시도 → 요청은 보이지만 내부 로직은 복잡함

## 주의사항

### ⚠️ 완벽한 보안은 없습니다
- 클라이언트 측 보안은 우회 가능
- 서버 측 인증/인가가 가장 중요
- 이 보안 기능은 **진입 장벽**을 높이는 역할

### ✅ 서버 측 보안이 핵심
현재 서버에서 이미 적용된 보안:
- JWT 토큰 인증
- 세션 관리
- 역할 기반 접근 제어 (마스터/매니저)
- 401/403 에러 처리

### 🔐 권장 사항
1. **HTTPS 필수** - 토큰 탈취 방지
2. **정기적인 토큰 갱신** - 세션 하이재킹 방지
3. **로그 모니터링** - 비정상 접근 감지
4. **강력한 비밀번호** - 마스터 계정 보호

## 문제 해결

### 정상적인 관리 작업이 어려울 때

개발/디버깅이 필요하다면 임시로 보안 해제:

1. **admin.html** 백업본 만들기:
```bash
cp /home/myno/바탕화면/myno/macroServer/public/admin.html admin.secure.html
```

2. 보안 기능 주석 처리 (개발 시)

3. 작업 완료 후 다시 활성화:
```bash
cp admin.secure.html admin.html
```
